# distill v0.1.0

> Any document → clean Markdown, in a single Go binary, built for LLM / RAG
> pipelines.

This is the first public release. The product is feature-complete for v1: a
single self-contained binary that converts the formats LLM teams actually
encounter — PDFs, Office documents, e-books, e-mail, archives, images, audio,
video — into structured Markdown, and exposes that capability through a CLI,
a concurrent batch converter, a secure HTTP service, and an MCP server.

---

## Install — one command

```bash
curl -fsSL https://raw.githubusercontent.com/MitudruDutta/distill/main/install.sh | bash
```

The installer detects your operating system and architecture, downloads the
matching asset from this release, and places the binary at
`~/.local/bin/distill`. It falls back to `go install` when no prebuilt asset
is available for your platform. To pin a specific version, set
`DISTILL_VERSION=v0.1.0` before running the command.

Alternative paths (`go install`, container, build from source) are described
in [`docs/install.md`](https://github.com/MitudruDutta/distill/blob/main/docs/install.md).

## Quick start

| Mode      | Command                                          | Purpose                                             |
| --------- | ------------------------------------------------ | --------------------------------------------------- |
| CLI       | `distill report.docx`                            | One-shot conversion, scripting, ad-hoc inspection.  |
| Batch     | `distill batch ./corpus --out-dir ./md`          | Concurrent directory-tree conversion for RAG ingest.|
| HTTP      | `distill serve --addr 127.0.0.1:8080`            | Backend microservice with `POST /convert`.          |
| MCP       | `distill mcp`                                    | Stdio Model Context Protocol server for AI agents.  |

## Highlights

### Generic Markdown shaping pass

Every converter's output flows through a shared post-processing step that:

- normalizes Unicode bullet glyphs (`•`, `▪`, `●`, …) and the entire
  Private Use Area (U+E000–U+F8FF, where many resume / CV fonts hide their
  bullets) to `- `, preserving indentation for nested lists;
- folds exotic Unicode whitespace (NBSP, em / en / ideographic spaces) to
  ASCII space and drops zero-width and BOM codepoints — visually identical,
  but materially fewer LLM tokens;
- splits implausibly long paragraphs at sentence boundaries, so a single
  multi-megabyte `<p>` becomes one sentence per line — well-formed input
  for chunked retrieval;
- preserves fenced code blocks byte-for-byte so JSON, YAML, and source code
  inside ` ``` ` blocks are never altered.

### PDF engine (`-tags pdfium`)

- Embeds PDFium as **WebAssembly via [wazero](https://wazero.io)** — pure
  Go, no cgo, no system library, no model download.
- **Font-size heading detection.** Rows materially larger than the page
  median become `# / ## / ###` based on relative ratio.
- **Layout-based table reconstruction.** Text rectangles are clustered into
  rows and columns; pages with stable column structure become Markdown
  tables. The decision is conservative — resumes and prose stay text by
  design.
- Falls back through `pdftotext` (poppler) → pure-Go reader → OCR for
  scanned PDFs, automatically.

### Office and OpenDocument structure

- **DOCX**: `Heading1..N` styles → `# .. ######`; `numPr` lists → bullet
  lists with indent; `w:tbl` → Markdown tables.
- **PPTX**: `## Slide N: <title>` (title-placeholder detection); `a:pPr lvl`
  → bullet indent; `a:tbl` inside graphic frames → Markdown tables.
- **ODT / ODS / ODP**: `text:outline-level` → headings; nested `text:list`
  → indented bullets; `table:table` → Markdown tables.

### Surfaces

- **MCP server** (`distill mcp`). Line-delimited JSON-RPC 2.0 over stdio
  with a single `convert` tool. Wire into Claude Desktop, Kiro CLI, or any
  MCP-compatible client in six lines of JSON. See
  [`docs/mcp.md`](https://github.com/MitudruDutta/distill/blob/main/docs/mcp.md);
  agent integration patterns in
  [`docs/agents.md`](https://github.com/MitudruDutta/distill/blob/main/docs/agents.md).
- **Batch mode** (`distill batch DIR --out-dir OUT`). Worker pool sized to
  `GOMAXPROCS`, mirrors input layout in the output directory,
  continue-on-error with a converted / failed summary, optional `--json`
  sidecars for downstream chunkers.
- **HTTP service** (`distill serve`). Secure-by-default: binds loopback,
  refuses non-loopback bind without `--token` (or `DISTILL_TOKEN`),
  constant-time auth compare, body-size cap (`--max-bytes`), read / write /
  idle timeouts. `POST /convert`, `GET /healthz`. Details:
  [`docs/serve.md`](https://github.com/MitudruDutta/distill/blob/main/docs/serve.md).

## Optional external tools (auto-detected)

distill stays self-contained out of the box. When these tools are present on
`PATH`, additional capabilities are enabled automatically; when they are
absent, the corresponding feature degrades gracefully.

| Tool                          | Capability                              |
| ----------------------------- | --------------------------------------- |
| `pdftotext` (poppler)         | Higher-fidelity PDF text extraction     |
| `pdftoppm` (poppler)          | Rasterizing scanned PDFs for OCR        |
| `tesseract`                   | Image OCR and scanned-PDF OCR           |
| `ffmpeg` / `ffprobe`          | Audio / video metadata extraction       |
| `whisper` (`openai-whisper`)  | Speech transcription                    |

## Roadmap

### Targeted for v0.1.1

- URL fetching (`distill <url>`) with strict SSRF protection.
- Plugin API: `--use-plugins` and `--list-plugins` for third-party converter
  registration.

### Targeted for a later release

- Outlook MSG and legacy XLS (BIFF) format support.
- Homebrew tap, Scoop manifest, and a published GHCR container image.

### Out of scope

- Cloud document-AI provider integrations.
- LLM-based image captioning.
- ML layout analysis.

distill is designed to remain a small, self-contained, single-binary
converter; capabilities beyond that scope belong in adjacent tools.
